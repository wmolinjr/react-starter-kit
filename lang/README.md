# Translations Directory

## Structure

```
lang/
├── {locale}/              # Nested files (EDIT HERE)
│   ├── admin/             # admin.* keys
│   ├── tenant/            # tenant.* keys
│   ├── enums/             # AUTO-GENERATED (don't edit!)
│   ├── permissions/       # AUTO-GENERATED (don't edit!)
│   └── *.json             # Other namespaces
├── {locale}.json          # AUTO-GENERATED flat file (don't edit!)
└── {locale}.json.bak      # Backup of original monolithic file
```

## Rules

### DO edit:
- Files in `lang/{locale}/admin/*.json`
- Files in `lang/{locale}/tenant/*.json`
- Files in `lang/{locale}/billing/*.json`
- Any other manual translation files

### DO NOT edit:
- `lang/{locale}.json` - Auto-generated by merging nested files
- `lang/{locale}/enums/*.json` - Auto-generated from PHP Enums
- `lang/{locale}/permissions/*.json` - Auto-generated from PHP Enums

## Enum Translation Keys

Enums use a hierarchical structure: `enums.{domain}.{type}.{value}`

| Enum | Key Pattern | File |
|------|-------------|------|
| AddonType | `enums.addon.type.*` | `enums/addon.json` |
| AddonStatus | `enums.addon.status.*` | `enums/addon.json` |
| BillingPeriod | `enums.billing.period.*` | `enums/billing.json` |
| PlanFeature | `enums.plan.feature.*` | `enums/plan.json` |
| PlanLimit | `enums.plan.limit.*` | `enums/plan.json` |
| TenantRole | `enums.tenant.role.*` | `enums/tenant.json` |
| TenantConfigKey | `enums.tenant.config.key.*` | `enums/tenant.json` |
| PermissionCategory | `enums.permission.category.*` | `enums/permission.json` |
| PermissionAction | `enums.permission.action.*` | `enums/permission.json` |
| BadgePreset | `enums.badge.preset.*` | `enums/badge.json` |

## Commands

```bash
# Regenerate types and merge translations
sail artisan types:generate

# Split monolithic file into nested structure
sail artisan i18n:split --dry-run
sail artisan i18n:split

# Find unused translation keys
sail artisan i18n:check-orphans
sail artisan i18n:check-orphans --fix
```

## Workflow

1. **Add/edit translations** in nested files (`lang/{locale}/namespace/*.json`)
2. **Run** `sail artisan types:generate` to merge and regenerate types
3. **Build** `sail npm run build` to include in production

## Source of Truth

- **Enum labels**: Defined in PHP Enums (`app/Enums/*.php`) via `name()` method
- **Manual translations**: Defined in nested JSON files
- **TypeScript types**: Auto-generated in `resources/js/types/translations.d.ts`

See [docs/I18N.md](../docs/I18N.md) for complete documentation.
