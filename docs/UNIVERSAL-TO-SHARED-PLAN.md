# Universal to Shared Namespace Refactoring Plan

## Overview

### What We Are Changing
Rename all occurrences of "Universal" to "Shared" across the codebase to better reflect the purpose of these components - they are shared between Central and Tenant contexts.

### Why
- **Clarity**: "Shared" better describes the intent - resources/models that work in both contexts
- **Consistency**: Aligns with industry terminology for shared/common modules
- **Stancl Tenancy**: Avoids confusion with Stancl's `UniversalRoutes` feature (which we don't use)

### Scope
- 8 directories to rename
- 53+ files requiring code changes
- 16 documentation files to update
- Wayfinder routes will be regenerated automatically

## Important Notes

### Middleware Group Name
The `universal` middleware group in `bootstrap/app.php` should be **kept as-is**:
```php
$middleware->group('universal', []);
```
This is required by `Stancl\Tenancy\Features\UniversalRoutes` and refers to Stancl's feature name, not our naming convention.

### Route Names vs Directory Names
We are changing:
- **Directories**: `Universal` → `Shared`
- **Route prefix**: `universal.settings.*` → `shared.settings.*`

But **keeping** the Stancl middleware name as `universal`.

### Wayfinder Regeneration
After PHP changes, Wayfinder **must** be regenerated to create new TypeScript files with correct paths. Old files must be manually deleted.
