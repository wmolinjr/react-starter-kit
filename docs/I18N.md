# Internacionalização (i18n)

Guia completo para utilização do sistema de internacionalização do projeto.

## Stack

- **Pacote**: [laravel-react-i18n](https://github.com/franciscohanna92/laravel-react-i18n) v2.0.5
- **Idioma Padrão**: Português Brasileiro (`pt_BR`)
- **Idioma Fallback**: Inglês (`en`)
- **Arquivos de Tradução**: `lang/pt_BR.json` e `lang/en.json`

## Configuração

### Vite Plugin

O plugin de i18n está configurado em `vite.config.ts`:

```typescript
import i18n from 'laravel-react-i18n/vite';

export default defineConfig({
    plugins: [
        // ... outros plugins
        i18n(),
    ],
});
```

### Provider React

O `LaravelReactI18nProvider` envolve a aplicação em `resources/js/app.tsx`:

```typescript
import { LaravelReactI18nProvider } from 'laravel-react-i18n';

root.render(
    <LaravelReactI18nProvider
        locale={props.initialPage.props.locale as string || 'pt_BR'}
        fallbackLocale="en"
        files={import.meta.glob('/lang/*.json')}
    >
        <App {...props} />
    </LaravelReactI18nProvider>
);
```

Para SSR (`resources/js/ssr.tsx`), usa-se `eager: true`:

```typescript
files={import.meta.glob('/lang/*.json', { eager: true })}
```

### Middleware Inertia

O locale é compartilhado via `HandleInertiaRequests.php`:

```php
public function share(Request $request): array
{
    return [
        // ...
        'locale' => app()->getLocale(),
        'availableLocales' => ['pt_BR', 'en'],
    ];
}
```

## Uso em Componentes

### Hook useLaravelReactI18n

```typescript
import { useLaravelReactI18n } from 'laravel-react-i18n';

export function MyComponent() {
    const { t } = useLaravelReactI18n();

    return (
        <div>
            <h1>{t('page.title')}</h1>
            <p>{t('page.description')}</p>
        </div>
    );
}
```

### Interpolação de Variáveis

```typescript
// JSON: "welcome": "Bem-vindo, :name!"
t('welcome', { name: 'João' })
// Resultado: "Bem-vindo, João!"
```

### Pluralização

```typescript
// JSON: "items": "{0} Nenhum item|{1} Um item|[2,*] :count itens"
t('items', { count: 0 })  // "Nenhum item"
t('items', { count: 1 })  // "Um item"
t('items', { count: 5 })  // "5 itens"
```

## Termos Técnicos em Inglês

**IMPORTANTE**: Mantemos termos técnicos em inglês para consistência:

- Dashboard (não "Painel")
- Role (não "Papel")
- Token (não "Ficha")
- Slug (não traduzir)
- Add-on (não traduzir)
- Owner, Admin, Member (roles)
- Feature (contexto técnico)

## Adicionando Novas Traduções

### 1. Adicione a Chave nos Arquivos JSON

**lang/pt_BR.json**:
```json
{
    "minha_pagina.titulo": "Meu Título",
    "minha_pagina.descricao": "Minha descrição em português"
}
```

**lang/en.json**:
```json
{
    "minha_pagina.titulo": "My Title",
    "minha_pagina.descricao": "My description in English"
}
```

### 2. Use no Componente

```typescript
import { useLaravelReactI18n } from 'laravel-react-i18n';

export function MinhaPage() {
    const { t } = useLaravelReactI18n();

    return (
        <>
            <h1>{t('minha_pagina.titulo')}</h1>

        </>
    );

```

### 3. Rebuild

```bash
sail npm run build
```

## Funções Disponíveis

O hook `useLaravelReactI18n()` retorna:

| Função | Descrição |
|--------|-----------|
| `t(key, replacements?)` | Traduz uma chave |
| `tChoice(key, count, replacements?)` | Tradução com pluralização |
| `currentLocale()` | Retorna locale atual |
| `setLocale(locale)` | Altera o locale |
| `getLocales()` | Lista locales disponíveis |
| `isLocale(locale)` | Verifica se é o locale atual |
| `loading` | Boolean indicando carregamento |

## Exemplo Completo: Nova Página

// resources/js/pages/tenant/reports/index.tsx

import { FileText } from 'lucide-react';

## Troca de Idioma em Runtime

```typescript
import { useLaravelReactI18n } from 'laravel-react-i18n';

export function LanguageSwitcher() {
    const { currentLocale, setLocale, getLocales } = useLaravelReactI18n();

    return (
        <select
            value={currentLocale()}
            onChange={(e) => setLocale(e.target.value)}
        >
            {getLocales().map((locale) => (
                <option key={locale} value={locale}>
                    {locale === 'pt_BR' ? 'Português' : 'English'}
                </option>
            ))}
        </select>
    );
}
```

## Troubleshooting

### Chaves Aparecendo no Lugar do Texto

1. Verifique se a chave existe em ambos os arquivos JSON
2. Verifique a ortografia da chave
3. Execute `sail npm run build` para reconstruir

### Erro "Invalid Hook Call"

Este erro ocorre quando há múltiplas instâncias do React. A solução está em `vite.config.ts`:

```typescript
export default defineConfig({
    resolve: {
        alias: {
            react: path.resolve('./node_modules/react'),
            'react-dom': path.resolve('./node_modules/react-dom'),
        },
    },
    // ...
});
```

### SSR Não Carrega Traduções

Certifique-se de usar `eager: true` no SSR:

```typescript
files={import.meta.glob('/lang/*.json', { eager: true })}
```

### Traduções Desatualizadas

```bash
# Limpar cache e rebuild
rm -rf node_modules/.vite
sail npm run build
```

## Boas Práticas

1. **Sempre adicione traduções em ambos os idiomas** (pt_BR e en)
2. **Use chaves descritivas** seguindo a convenção de nomenclatura
3. **Agrupe chaves por área/página** para facilitar manutenção
4. **Mantenha termos técnicos em inglês** conforme lista acima
5. **Teste após adicionar novas chaves** para garantir que aparecem corretamente
6. **Evite strings hardcoded** - sempre use o sistema de i18n

## Traduções no Banco de Dados (Database i18n)

Para dados de sistema como Plans, Addons e Roles, utilizamos **Spatie Laravel Translatable**.

### Models com Traduções

| Model | Campos Traduzíveis |
|-------|-------------------|
| `Plan` | name, description |
| `Addon` | name, description, unit_label |
| `Role` | display_name, description |

### Uso

```php
// Obter tradução para locale atual (via __get do Spatie)
$plan->name;                              // Retorna tradução do locale atual
$plan->description;

// Obter tradução para locale específico
$plan->getTranslation('name', 'en');      // "Professional"
$plan->getTranslation('name', 'pt_BR');   // "Profissional"

// Obter todas as traduções
$plan->getTranslations('name');           // ['en' => 'Professional', 'pt_BR' => 'Profissional']

// Definir tradução
$plan->setTranslation('name', 'en', 'Premium');
$plan->setTranslation('name', 'pt_BR', 'Premium');
$plan->save();
```

### Seeder com Traduções

```php
// database/seeders/PlanSeeder.php
Plan::create([
    'slug' => 'professional',
    'name' => [
        'en' => 'Professional',
        'pt_BR' => 'Profissional',
    ],
    'description' => [
        'en' => 'For growing teams',
        'pt_BR' => 'Para equipes em crescimento',
    ],
    // ...
]);
```

### API Resources

Os Resources usam o método helper `trans()` do BaseResource:

```php
// app/Http/Resources/Central/PlanResource.php
public function toArray($request): array
{
    return [
        'name' => $this->trans('name'),         // Tradução do locale atual
        'description' => $this->trans('description'),
        'all_names' => $this->translations('name'), // Todas as traduções
    ];
}
```
