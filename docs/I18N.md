# Internacionalização (i18n)

Guia completo para utilização do sistema de internacionalização do projeto.

## Stack

- **Pacote**: [laravel-react-i18n](https://github.com/franciscohanna92/laravel-react-i18n) v2.0.5
- **Idioma Padrão**: Português Brasileiro (`pt_BR`)
- **Idioma Fallback**: Inglês (`en`)
- **Arquivos de Tradução**: `lang/pt_BR.json` e `lang/en.json`

## Configuração

### Vite Plugin

O plugin de i18n está configurado em `vite.config.ts`:

```typescript
import i18n from 'laravel-react-i18n/vite';

export default defineConfig({
    plugins: [
        // ... outros plugins
        i18n(),
    ],
});
```

### Provider React

O `LaravelReactI18nProvider` envolve a aplicação em `resources/js/app.tsx`:

```typescript
import { LaravelReactI18nProvider } from 'laravel-react-i18n';

root.render(
    <LaravelReactI18nProvider
        locale={props.initialPage.props.locale as string || 'pt_BR'}
        fallbackLocale="en"
        files={import.meta.glob('/lang/*.json')}
    >
        <App {...props} />
    </LaravelReactI18nProvider>
);
```

Para SSR (`resources/js/ssr.tsx`), usa-se `eager: true`:

```typescript
files={import.meta.glob('/lang/*.json', { eager: true })}
```

### Middleware Inertia

O locale é compartilhado via `HandleInertiaRequests.php`:

```php
public function share(Request $request): array
{
    return [
        // ...
        'locale' => app()->getLocale(),
        'availableLocales' => ['pt_BR', 'en'],
    ];
}
```

## Uso em Componentes

### Hook useLaravelReactI18n

```typescript
import { useLaravelReactI18n } from 'laravel-react-i18n';

export function MyComponent() {
    const { t } = useLaravelReactI18n();

    return (
        <div>
            <h1>{t('page.title')}</h1>
            <p>{t('page.description')}</p>
        </div>
    );
}
```

### Interpolação de Variáveis

```typescript
// JSON: "welcome": "Bem-vindo, :name!"
t('welcome', { name: 'João' })
// Resultado: "Bem-vindo, João!"
```

### Pluralização

```typescript
// JSON: "items": "{0} Nenhum item|{1} Um item|[2,*] :count itens"
t('items', { count: 0 })  // "Nenhum item"
t('items', { count: 1 })  // "Um item"
t('items', { count: 5 })  // "5 itens"
```

## Termos Técnicos em Inglês

**IMPORTANTE**: Mantemos termos técnicos em inglês para consistência:

- Dashboard (não "Painel")
- Role (não "Papel")
- Token (não "Ficha")
- Slug (não traduzir)
- Add-on (não traduzir)
- Owner, Admin, Member (roles)
- Feature (contexto técnico)

## Funções Disponíveis

O hook `useLaravelReactI18n()` retorna:

| Função | Descrição |
|--------|-----------|
| `t(key, replacements?)` | Traduz uma chave |
| `tChoice(key, count, replacements?)` | Tradução com pluralização |
| `currentLocale()` | Retorna locale atual |
| `setLocale(locale)` | Altera o locale |
| `getLocales()` | Lista locales disponíveis |
| `isLocale(locale)` | Verifica se é o locale atual |
| `loading` | Boolean indicando carregamento |

## Troca de Idioma em Runtime

```typescript
import { useLaravelReactI18n } from 'laravel-react-i18n';

export function LanguageSwitcher() {
    const { currentLocale, setLocale, getLocales } = useLaravelReactI18n();

    return (
        <select
            value={currentLocale()}
            onChange={(e) => setLocale(e.target.value)}
        >
            {getLocales().map((locale) => (
                <option key={locale} value={locale}>
                    {locale === 'pt_BR' ? 'Português' : 'English'}
                </option>
            ))}
        </select>
    );
}
```

## Troubleshooting

### Chaves Aparecendo no Lugar do Texto

1. Verifique se a chave existe em ambos os arquivos JSON
2. Verifique a ortografia da chave
3. Execute `sail npm run build` para reconstruir

### Erro "Invalid Hook Call"

Este erro ocorre quando há múltiplas instâncias do React. A solução está em `vite.config.ts`:

```typescript
export default defineConfig({
    resolve: {
        alias: {
            react: path.resolve('./node_modules/react'),
            'react-dom': path.resolve('./node_modules/react-dom'),
        },
    },
    // ...
});
```

### SSR Não Carrega Traduções

Certifique-se de usar `eager: true` no SSR:

```typescript
files={import.meta.glob('/lang/*.json', { eager: true })}
```

### Traduções Desatualizadas

```bash
# Limpar cache e rebuild
rm -rf node_modules/.vite
sail npm run build
```

## Boas Práticas

1. **Sempre adicione traduções em ambos os idiomas** (pt_BR e en)
2. **Use chaves descritivas** seguindo a convenção de nomenclatura
3. **Agrupe chaves por área/página** para facilitar manutenção
4. **Mantenha termos técnicos em inglês** conforme lista acima
5. **Teste após adicionar novas chaves** para garantir que aparecem corretamente
6. **Evite strings hardcoded** - sempre use o sistema de i18n

## Traduções no Banco de Dados (Database i18n)

Para dados de sistema como Features, Limits, Plans e Addons, utilizamos **Spatie Laravel Translatable** com suporte a **Tenant Overrides** para white-label.

### Arquitetura

```
┌─────────────────────────────────────────────────────────────────┐
│                    TRANSLATION RESOLUTION                        │
│                                                                  │
│   1. Tenant Override (tenant_translation_overrides)              │
│              ↓ não encontrado                                    │
│   2. Base Translation (Spatie Translatable no model)             │
│              ↓ não encontrado                                    │
│   3. Fallback Locale (en)                                        │
└─────────────────────────────────────────────────────────────────┘
```

### Models com Traduções

| Model | Campos Traduzíveis | Customizable |
|-------|-------------------|--------------|
| `FeatureDefinition` | name, description | Sim (flag) |
| `LimitDefinition` | name, description, unit_label | Sim (flag) |

### Uso Básico

```php
// Obter tradução (usa locale da app + tenant override se existir)
$limit->trans('name');                    // "User Seats" ou "Vagas de Usuário"
$limit->trans('name', 'pt_BR');           // Força pt_BR

// Obter tradução base (ignora tenant override)
$limit->getTranslation('name', 'en');     // "User Seats"
$limit->getTranslations('name');          // ['en' => '...', 'pt_BR' => '...']

// Verificar se tem override do tenant
$limit->hasTenantOverride('name');        // true/false

// Verificar se campo permite customização
$limit->is_customizable;                  // true/false
```

### Tenant Overrides (White-Label)

```php
// Definir override para o tenant atual
$limit->setTenantTranslation('name', 'en', 'Team Members');
$limit->setTenantTranslation('name', 'pt_BR', 'Membros da Equipe');

// Definir múltiplas traduções de uma vez
$limit->setTenantTranslations('name', [
    'en' => 'Team Members',
    'pt_BR' => 'Membros da Equipe',
]);

// Remover override específico
$limit->removeTenantTranslation('name', 'en');

// Remover todos os overrides do campo
$limit->removeTenantTranslation('name');

// Obter dados completos para frontend
$limit->getTranslationData('name');
// Returns: [
//     'base' => ['en' => '...', 'pt_BR' => '...'],
//     'override' => ['en' => '...', 'pt_BR' => '...'],
//     'current' => [...],  // merged
//     'has_override' => true,
//     'is_customizable' => true,
// ]
```

### Seeder com Traduções

```php
// database/seeders/DefinitionsSeeder.php
[
    'key' => 'users',
    'name' => [
        'en' => 'User Seats',
        'pt_BR' => 'Vagas de Usuário',
    ],
    'description' => [
        'en' => 'Maximum number of team members',
        'pt_BR' => 'Número máximo de membros da equipe',
    ],
    'is_customizable' => true,  // permite override por tenant
]
```

### Controle de Customização

O campo `is_customizable` controla se tenants podem fazer override das traduções:

| `is_customizable` | Comportamento |
|-------------------|---------------|
| `true` | Tenant pode customizar (white-label) |
| `false` | Tradução fixa, não permite override |

**Exemplos de campos NÃO customizáveis:**
- Termos técnicos: SSO, White Label, API
- Métricas fixas: Log Retention, Max File Size

### Controller para Frontend

```php
public function index(): Response
{
    $limits = LimitDefinition::all()->map(
        fn ($limit) => $limit->toTranslatedArray(tenant('id'))
    );

    return Inertia::render('limits/index', [
        'limits' => $limits,
    ]);
}
```

### Tabela de Overrides

```
tenant_translation_overrides
├── id
├── tenant_id (FK)
├── translatable_type (App\Models\LimitDefinition)
├── translatable_id (1)
├── field (name)
├── translations ({"en": "Custom Name", "pt_BR": "Nome Custom"})
└── timestamps
```

### Cache

Os overrides são cacheados com tags para performance:
- Tag: `tenant_translations`
- Tag: `tenant:{id}`
- TTL: 1 hora

O cache é limpo automaticamente ao modificar overrides.
